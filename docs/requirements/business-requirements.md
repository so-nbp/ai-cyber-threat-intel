## BR-1: マルチソース脅威情報収集 ✅

AI に関連する脅威情報を複数の公開ソースから自動収集する。

| 項目 | 内容 |
|---|---|
| 優先度 | P0（必須） |
| 対象ユーザー | 全ペルソナ |
| 成功指標 | 6つ以上のソースから日次自動収集 |
| 現状 | 6ソース実装済み（NVD, CISA KEV, GitHub Advisory, arXiv, RSS, OTX） |

---

## BR-2: AI脅威の自動分類 🔨

収集した情報を脅威カテゴリ・深刻度・信頼度で自動分類する。

| 項目 | 内容 |
|---|---|
| 優先度 | P0（必須） |
| 対象ユーザー | CTIアナリスト, CISO |
| 成功指標 | 全アイテムが7軸カテゴリで自動分類される |
| 現状 | キーワードベースで4カテゴリ実装済み。taxonomy拡張（3カテゴリ追加）が必要 |

**分類カテゴリ一覧（7軸）:**

| カテゴリ | 定義 | 実装状況 |
|---|---|---|
| AI-as-Target | AI システム自体への攻撃 | ✅ 実装済み |
| AI-as-Weapon | AI が攻撃ツールとして悪用される | ✅ 実装済み |
| AI-Enabled | AI インフラ・フレームワークの脆弱性 | ✅ 実装済み |
| AI-Adjacent | AI エコシステム周辺基盤への脅威 | ✅ 実装済み |
| AI-Physical | AI 制御物理系システムへの脅威 | 📋 未実装（taxonomy拡張必要） |
| AI-Supply-Chain | AIモデル・データのサプライチェーン汚染 | 📋 未実装（taxonomy拡張必要） |
| AI-Agentic | 自律AIエージェントへの脅威 | 📋 未実装（taxonomy拡張必要） |
