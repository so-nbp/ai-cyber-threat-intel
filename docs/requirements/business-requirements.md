| AI-Enabled | AI インフラ・フレームワークの脆弱性 | ✅ 実装済み |

| AI-Supply-Chain | AIモデル・データのサプライチェーン汚染 | 📋 未実装（taxonomy拡張必要） |
| AI-Agentic | 自律AIエージェントへの脅威 | 📋 未実装（taxonomy拡張必要） |

### BR-4-1: 概要ダッシュボード（ホーム画面）

**目的**: 一目で現在のAI脅威の全体像をつかむ

**対象ユーザー**: CISO（週次リスク把握）、CTIアナリスト（毎朝の状況確認）

**必須要素:**

| 要素 | 内容 | 根拠 |
|---|---|---|
| KPIカード | 総収集件数 / Critical+High件数 / AI関連割合 / 過去24h新着件数 | 即時の状況把握 |
| カテゴリ別内訳（ドーナツグラフ） | 7軸カテゴリの件数比率 | どの脅威領域が多いかの俯瞰 |
| 深刻度分布（棒グラフ） | Critical / High / Medium / Low / Unknown の件数 | リスク水準の全体把握 |
| 収集件数トレンド（折れ線グラフ） | 過去30日間の日次収集件数推移 | 脅威情報量の変化検知 |
| ソース別内訳 | NVD / CISA KEV / GitHub Advisory / arXiv / RSS / OTX 別の件数 | 情報源の信頼性確認 |

**成功基準:**
- ページ読み込み後 3秒以内に全KPIが表示されること
- どのペルソナでも説明なしに意味を理解できること（直感的なラベルと色使い）

### BR-4-2: 脅威一覧ページ

**目的**: 蓄積されたデータを効率よく絞り込み・探索する

**対象ユーザー**: CTIアナリスト（日常的な調査起点）、AI/MLエンジニア（特定製品の脆弱性確認）

**必須要素:**

| 要素 | 内容 |
|---|---|
| 一覧テーブル | タイトル / 深刻度 / カテゴリ / ソース / 公開日時 を列表示 |
| フィルタバー | カテゴリ / 深刻度 / ソース / 期間（日付範囲）/ AI関連フラグ で絞り込み |
| テキスト検索 | タイトル・説明文の全文検索 |
| ソート | 公開日時・深刻度・カテゴリで並び替え |
| ページネーション | 1ページ 20〜50件表示、ページ移動可能 |
| 詳細リンク | 各行クリックで詳細ページへ遷移 |

**成功基準:**
- フィルタ変更後 2秒以内に結果が更新されること
- 深刻度を色・バッジで視覚的に区別できること（Critical=赤、High=オレンジ等）

### BR-4-3: 脅威詳細ページ

**目的**: 個々の脅威情報の全内容を確認し、対応判断に必要な情報を得る

**対象ユーザー**: CTIアナリスト（インシデント対応時の詳細調査）、AI/MLエンジニア（脆弱性の影響範囲確認）

**必須要素:**

| 要素 | 内容 |
|---|---|
| ヘッダー | タイトル / 深刻度バッジ / カテゴリタグ / ソース名 |
| メタ情報 | 公開日時 / 収集日時 / CVSS スコア / 信頼度 / AI関連フラグ |
| 本文 | 説明文（全文表示） |
| CVE情報 | 関連CVE ID（複数の場合は一覧） |
| 影響製品 | affected_products（製品名・バージョン） |
| タグ・キーワード | tags / keywords のバッジ表示 |
| 参照リンク | references / source_url への外部リンク（新しいタブで開く） |
| ナビゲーション | 「一覧に戻る」ボタン / 前の脅威・次の脅威ボタン |

**成功基準:**
- 外部ソース（source_url）への1クリックアクセスが可能なこと
- 前後の脅威アイテムへ移動できること（調査の連続性を保つ）
- 詳細ページのURLが一意であること（共有・ブックマーク可能）

## BR-9: 産業セクター別脅威ダッシュボード 📋

各産業セクターのユーザーが、自分のセクターに関連するAI脅威情報を一目で把握できるようにする。

| 項目 | 内容 |
|---|---|
| 優先度 | P1（次フェーズ） |
| 対象ユーザー | CISO（全セクター）、CTIアナリスト、各セクターのセキュリティ担当者 |
| 成功指標 | 12セクターそれぞれの脅威状況をダッシュボードで閲覧可能。セクター別の件数・深刻度・最新脅威が表示される |
| 現状 | `AffectedSector` enum と `affected_sectors` DB フィールドは実装済み。ダッシュボードのセクター別表示は未実装 |

### 背景・なぜ必要か

現状のダッシュボードは「脅威カテゴリ別（AI-as-Target等）」でサマリしているが、これは**攻撃手法の分類**であり、**誰が被害を受けるか**という視点がない。

金融機関のCISO、医療機関のセキュリティ担当者、エネルギー企業のSOCアナリストは、それぞれ「自分の業界に向けられた脅威」を最優先で把握したい。セクター別ビューはこのニーズに応える。

また、このプロダクトをWebサービスとして複数セクターの組織向けに公開する際、セクター別のパーソナライズが差別化の重要な軸となる。

### 採用するセクター区分（12セクター）

CISA（米国）・NISC（日本）・NIS2指令（EU）の3つの重要インフラ標準に共通して登場するセクターを基準に策定。詳細は ADR-006 を参照。

| コード | 日本語名 | スコープ | 対応ISAC |
|---|---|---|---|
| ENERGY | エネルギー | 電力・ガス・石油・水素 | E-ISAC |
| FINANCIAL | 金融 | 銀行・証券・保険・金融市場インフラ・クレジット | FS-ISAC |
| HEALTHCARE | 医療・ヘルスケア | 医療機関・製薬・医療機器・公衆衛生 | H-ISAC |
| TELECOMMUNICATIONS | 通信・情報通信 | 通信キャリア・DNS・CDN・インターネット交換 | Comm-ISAC |
| TRANSPORTATION | 交通・物流 | 航空・鉄道・道路・海運・港湾・空港・物流 | A-ISAC / MTS-ISAC |
| GOVERNMENT | 政府・行政 | 国・地方行政機関・政府施設・緊急サービス | MS-ISAC |
| DEFENSE | 防衛・宇宙 | 防衛産業・軍事・宇宙インフラ | NDISAC / Space ISAC |
| TECHNOLOGY | 情報技術・IT | IT産業・クラウド・ソフトウェア・MSP/MSSP | IT-ISAC |
| MANUFACTURING | 製造業 | 重要製造業・化学・自動車・産業機械 | MFG-ISAC |
| EDUCATION | 教育・研究 | 大学・研究機関・公教育 | REN-ISAC |
| GENERAL | 小売・食品・農業・その他 | 小売・食品製造・農業・商業施設・上下水道など | Food & Ag ISAC |
| UNKNOWN | 分類不明 | セクター未特定 | - |

### BR-9-1: ダッシュボードホームへのセクターサマリー追加

**目的**: 全セクターの脅威状況を比較できる俯瞰ビュー

**対象ユーザー**: CISO（経営判断のため横断的な状況把握が必要）、セキュリティコンサルタント（複数クライアントのセクターを比較）

**必須要素:**

| 要素 | 内容 | 根拠 |
|---|---|---|
| セクター別件数ヒートマップ or 横棒グラフ | 12セクター × Critical/High件数を可視化 | どのセクターへの脅威が多いかを一目で把握 |
| セクター別リスクランキング | Critical+High件数で上位5セクターを表示 | 優先的に注目すべきセクターの識別 |
| 最新脅威バイセクター | セクターごとの直近7日間の新着件数 | タイムリーな動向把握 |

**成功基準:**
- ページ読み込み後3秒以内に表示されること
- 各セクターのリスクレベルが色（例: 件数が多いほど濃い赤）で直感的に理解できること

### BR-9-2: セクター別詳細ビュー（新規ページ）

**目的**: 特定セクターのユーザーが自分のセクターに絞って脅威を深掘りできる専用ページ

**対象ユーザー**: 各セクターの CISO・CTIアナリスト・セキュリティ担当者

**必須要素:**

| 要素 | 内容 |
|---|---|
| セクター選択UI | 12セクターからドロップダウンまたはカードで選択 |
| セクター別KPIカード | 選択セクターの総件数 / Critical+High件数 / 過去7日間新着 / 最多脅威カテゴリ |
| 深刻度分布グラフ | 選択セクターにおける Critical/High/Medium/Low の内訳棒グラフ |
| 脅威カテゴリ分布 | 選択セクターで多い AI 脅威タイプ（AI-as-Target等）のドーナツグラフ |
| 直近の脅威一覧 | 選択セクターの最新10〜20件（タイトル/深刻度/日時/詳細リンク付き） |

**成功基準:**
- セクター切り替え後2秒以内に全KPIが更新されること
- セクターのURLが一意（例: `?sector=FINANCIAL`）で共有・ブックマーク可能なこと

### BR-9-3: 脅威一覧ページへのセクターフィルター追加

**目的**: 既存の脅威一覧（BR-4-2）にセクターフィルターを追加する

**対象ユーザー**: CTIアナリスト（セクター横断での脅威調査）

**必須要素:**
- 既存フィルターバーに「セクター」項目を追加（単一または複数選択）
- セクターフィルターと既存フィルター（カテゴリ/深刻度/ソース）の組み合わせが可能なこと

**成功基準:**
- セクターフィルター適用後2秒以内に一覧が更新されること
