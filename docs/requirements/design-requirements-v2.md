## 1. デザイン原則

| # | 原則 | 説明 |
|---|---|---|
| D1 | **状況認識の即時性** | ダッシュボードを開いた瞬間に「今、何が起きているか」がわかる |
| D2 | **深刻度の視覚的優先** | Critical → High → Medium → Low の順に視覚的に目立つ |
| D3 | **ドリルダウン導線** | 概要 → 詳細 → 原文 への段階的な情報掘り下げが自然にできる |
| D4 | **アクション指向** | 情報を見たら「次に何をすべきか」が明示されている |
| D5 | **ダークテーマ基調** | SOC環境での長時間利用に適した暗色ベースのUI |
| D6 | **情報密度の最適化** | 一画面に十分な情報を表示しつつ、視認性を損なわない |

#### セクター画面のコンポーネント仕様

**セクター別ヒートマップ（§5.6）**
- セルの背景色: 深刻度カラー（§3準拠）を件数に応じた不透明度（0.1〜1.0）で表現
- 件数ゼロのセルは `#21262D`（背景セカンダリ）で表示
- 前月比: 増加 `▲` は `#FF4444`（赤）、減少 `▼` は `#3FB950`（緑）、横ばい `─` は `#8B949E`（グレー）
- 行ホバー時: 背景 `#1C2633` でハイライト、カーソルを pointer に変更
- 行クリック: セクター別ドリルダウン画面に遷移

**セクター別トレンドグラフ** — §5.5 と同一コンポーネントを再利用。セクターごとに異なる線色（アクセント色系統から自動割当）。凡例クリックで表示/非表示を切替。

**セクター別カテゴリ構成グラフ（§5.7）** — §3 脅威カテゴリカラー準拠の横棒グラフ。セクター選択ドロップダウンで切替。

### 5.2 KPI カード（直近24時間）

```
┌──────────────────────┐
│  アラート概況（24h）   │
│                      │
│    28                │  ← 大きい数値（36-48px, Bold）
```

カード仕様:
- 背景: `#161B22`
- ボーダー: `#30363D` 1px
- 角丸: 8px
- パディング: 20px
- ホバー時: ボーダー `#58A6FF` でハイライト

### 5.3 データテーブル

```
┌────────┬────────────┬───────────┬──────────────────────┬──────────┐
│ 深刻度  │ カテゴリ    │ ソース     │ タイトル              │ 日時     │
├────────┼────────────┼───────────┼──────────────────────┼──────────┤
│ ■ CRIT │ AI-Target  │ NVD       │ CVE-2024-XXXX ...    │ 2h ago   │
├────────┼────────────┼───────────┼──────────────────────┼──────────┤
│ ■ HIGH │ AI-Enabled │ GitHub    │ GHSA-XXXX ...        │ 5h ago   │
└────────┴────────────┴───────────┴──────────────────────┴──────────┘
```

テーブル仕様:
- ヘッダー背景: `#21262D`
- 奇数行: `#161B22`, 偶数行: `#0D1117`
- ホバー行: 背景 `#1C2633`
- 行クリックで詳細画面に遷移
- ソート可能（各カラムヘッダークリック）
- フィルター（深刻度、カテゴリ、ソース、期間、セクター）

### 5.4 ATT&CK マトリクス表示

- 各戦術（Tactic）を列、各技術（Technique）をセルとして表示
- セルの色の濃淡で検知頻度/該当件数を表現（ヒートマップ）
- セルクリックで該当する脅威アイテム一覧にドリルダウン

### 5.5 時系列グラフ

- ライブラリ: Recharts（React）または Plotly
- X軸: 日付, Y軸: 件数
- 深刻度別の折れ線を重ねて表示（マルチライン）
- ツールチップでその日の内訳を表示
- 期間切替: 7日 / 30日 / 90日
- セクター別画面でも同一コンポーネントを再利用

### 5.6 セクター別ヒートマップテーブル【v2追加】

- 行: 産業セクター（脅威件数降順でソート）
- 列: Critical / High / Medium / Low / 合計 / 前月比
- セルの背景色: 該当する深刻度カラーを件数に応じた不透明度（0.1〜1.0）で表現
- 件数ゼロのセル: `#21262D`（背景セカンダリ）
- 前月比: `▲` 増加（赤）, `▼` 減少（緑）, `─` 横ばい（グレー）
- 行ホバー: 背景 `#1C2633`、カーソル pointer

## 9. 技術選定（推奨）

| 区分 | 推奨技術 | 理由 |
|---|---|---|
| Phase 2（MVP UI） | Streamlit | Python のみで構築可能。プロトタイプに最適 |
| Phase 3（本格UI） | React + Tailwind CSS | コンポーネントベースで拡張性が高い |
| グラフ | Recharts / Plotly | インタラクティブな時系列/ヒートマップ |
| アイコン | Lucide React | 軽量で統一感のあるアイコンセット |
| フォント | Inter + Noto Sans JP | 英日混在に対応、可読性が高い |
