## 1. サービスコンセプト

### PwCのアプローチから読み取れる本質

PwCのサービスは「IoC配信型（現在の主流）」から「インテリジェンス主導型」への転換を提唱している。
その核心は以下の3点:

1. **脅威アクター起点の分析**: 単なる脆弱性リストではなく、「誰が」「なぜ」「どう攻撃するか」を起点にする
2. **個社カスタマイズ**: 業種・地域・システム環境に基づき、脅威を絞り込む
3. **能動的防御（アクティブディフェンス）**: 過去の検知から次の攻撃を予測し、先手を打つ

### 本プロジェクトへの示唆

現在の MVP は「情報収集・分類」に留まっている。
PwC レベルのサービスに到達するには、以下の進化が必要:

```
現在地                →  目指すべき姿
─────────────────────────────────────────────
情報の収集・蓄積       →  脅威アクター単位の追跡
キーワード分類         →  TTPs ベースの構造化分析
全データ一律表示       →  ユーザー環境に応じた絞り込み
全セクター一律         →  産業セクター別の脅威可視化
過去の記録             →  次の攻撃の予測
CLI テキスト出力       →  リアルタイムダッシュボード
```

## 2. ビジネス要件一覧

### BR-A: 脅威アクター追跡・プロファイリング

| 項目 | 内容 |
|---|---|
| PwCの対応機能 | 数百の脅威アクターを常時観測、標的国・業界・TTPs を分析 |
| 要件概要 | 脅威アクターを識別・追跡し、動機・標的・手口をプロファイルとして蓄積する |
| 具体的な機能 | - 脅威アクターのデータベース（名称、別名、帰属国、動機、標的業種/地域） |
|  | - アクターと攻撃手法（TTPs）の紐付け |
|  | - アクターの活動タイムライン（いつ、どこで、何をしたか） |
| 優先度 | P1（差別化の核） |
| 現状ギャップ | 未実装。現在は脆弱性・記事単位でのデータ収集のみ |

### BR-B: TTPs（戦術・技術・手順）の構造化管理

| 項目 | 内容 |
|---|---|
| PwCの対応機能 | MITRE ATT&CK マトリクス上での攻撃手法マッピング・可視化 |
| 要件概要 | 収集した脅威情報を MITRE ATT&CK / ATLAS の技術 ID に紐付け、攻撃手法を構造的に管理する |
| 具体的な機能 | - ATT&CK テクニック ID の自動/手動マッピング |
|  | - ATT&CK マトリクスのヒートマップ表示（どの手法が多いか） |
|  | - テクニック → 検知ルール → 対応手順の連鎖管理 |
| 優先度 | P1 |
| 現状ギャップ | データモデルに `atlas_techniques`, `attck_techniques` フィールドは存在するが未活用 |

### BR-C: 個社カスタマイズ（業種・地域・環境ベースの絞り込み）

| 項目 | 内容 |
|---|---|
| PwCの対応機能 | 貴社の業種や活動地域などのビジネス特性を踏まえて脅威アクターを絞り込み |
| 要件概要 | ユーザーが自社のプロファイル（業種、地域、使用技術）を登録し、関連する脅威のみを優先表示する |
| 具体的な機能 | - ユーザープロファイル設定（業種、国/地域、使用ML基盤/ライブラリ） |
|  | - プロファイルに基づく脅威の自動フィルタリング・スコアリング |
|  | - 「自社に関連する脅威」ビューの提供 |
| 優先度 | P2 |
| 現状ギャップ | 未実装。全ユーザーに同一データを表示 |
| 関連要件 | BR-K（産業セクター別分析）が基盤となる |

### BR-D: 専用脅威検知エンジン（検知ルール管理）

| 項目 | 内容 |
|---|---|
| PwCの対応機能 | 数百〜数千の検知ロジックを実装した貴社専用の脅威検知エンジン |
| 要件概要 | 脅威情報から SIEM 検知ルール（Sigma / YARA / Snort 等）を生成・管理する |
| 具体的な機能 | - 脅威情報から検知ルールの自動/半自動生成 |
|  | - ルールのエクスポート（Sigma, YARA, Snort, Splunk SPL 等） |
|  | - ルールの有効性追跡（検知件数、誤検知率） |
| 優先度 | P2 |
| 現状ギャップ | 未実装 |

### BR-E: IoC（Indicator of Compromise）管理・配信

| 項目 | 内容 |
|---|---|
| PwCの対応機能 | 脅威アクターに関する IoC のダウンロードが可能 |
| 要件概要 | 収集した IoC（IP, ドメイン, ハッシュ, URL 等）を構造化し、ダウンロード・API 配信する |
| 具体的な機能 | - IoC の自動抽出（脅威情報本文やレポートから） |
|  | - IoC 種別ごとの管理（IP, ドメイン, ファイルハッシュ, URL, メールアドレス） |
|  | - CSV / STIX / JSON 形式でのエクスポート |
|  | - SIEM / EDR への自動フィード |
| 優先度 | P1 |
| 現状ギャップ | OTX コレクターで一部 IoC を取得しているが、独立した管理機能はない |

### BR-F: アクティブディフェンス（攻撃予測）

| 項目 | 内容 |
|---|---|
| PwCの対応機能 | 過去と直近に検知した攻撃から次の攻撃を予想 |
| 要件概要 | 過去の攻撃パターンと脅威アクターのTTPsから、次に発生しうる攻撃を予測する |
| 具体的な機能 | - 攻撃チェーン分析（Kill Chain / ATT&CK 上での進行予測） |
|  | - 脅威アクターの過去パターンに基づく「次の手」の推定 |
|  | - 予測に基づく防御推奨アクションの提示 |
| 優先度 | P3（高度な分析。後期フェーズ） |
| 現状ギャップ | 未実装 |

### BR-G: リアルタイムダッシュボード

| 項目 | 内容 |
|---|---|
| PwCの対応機能 | アラート概況（直近24時間）、深刻度内訳、脅威の内容と予想、時系列表示 |
| 要件概要 | Web ブラウザ上でリアルタイムに脅威状況を把握できるダッシュボードを提供する |
| 具体的な機能 | - 直近24時間のアラート概況（件数、深刻度別内訳） |
|  | - 脅威の内容と予想（ATT&CK マトリクス表示） |
|  | - 脅威アクター候補の特定・表示 |
|  | - 時系列トレンドグラフ（30日 / 90日） |
|  | - 活動中の脅威アクター一覧と IoC ダウンロード |
|  | - グローバルナビに「セクター別」タブを配置【v2追加】 |
|  | - グローバルフィルターに「セクター」選択肢を追加【v2追加】 |
| 優先度 | P1 |
| 現状ギャップ | 未実装。CLI のみ |
| デザイン仕様 | design-requirements-v2.md §2.1, §6 |

### BR-H: 地政学リスク連関分析

| 項目 | 内容 |
|---|---|
| PwCの対応機能 | 地政学イベント・GPR・インシデントレポート・脆弱性レポートを連関させてセキュリティ要件を抽出 |
| 要件概要 | サイバー脅威を地政学的な文脈と結びつけて分析する |
| 具体的な機能 | - 地政学イベントのタイムライン（国際紛争、制裁、選挙等） |
|  | - イベントとサイバー攻撃の相関分析 |
|  | - 地域別脅威ヒートマップ |
| 優先度 | P3（将来） |
| 現状ギャップ | 未実装 |

### BR-I: 対応手順（プレイブック）管理

| 項目 | 内容 |
|---|---|
| PwCの対応機能 | 検知ルールごとの対応手順書を提供、検知イベント単位で対応手順を自動表示 |
| 要件概要 | 脅威検知時の対応手順を体系的に管理し、自動的に提示する |
| 具体的な機能 | - 脅威カテゴリ / テクニック ID ごとの対応手順テンプレート |
|  | - 検知イベントと対応手順の自動紐付け |
|  | - 対応手順の Markdown / PDF エクスポート |
| 優先度 | P2 |
| 現状ギャップ | 未実装 |

### BR-J: レポーティング・情報共有

| 項目 | 内容 |
|---|---|
| PwCの対応機能 | 脅威トレンドレポート、攻撃予想、脅威アクター分析レポート |
| 要件概要 | 定期レポートおよびアドホックレポートを自動/半自動で生成する |
| 具体的な機能 | - 日次/週次/月次の脅威サマリーレポート自動生成 |
|  | - エグゼクティブ向けサマリー（CISO / 経営層向け） |
|  | - STIX/TAXII 形式での機械可読共有 |
|  | - メール / Slack 通知 |
| 優先度 | P1 |
| 現状ギャップ | CLI の `summary` コマンドのみ。自動生成・配信機能なし |

### BR-K: 産業セクター別脅威分析【v2新規追加】

| 項目 | 内容 |
|---|---|
| PwCの対応機能 | 業種や活動地域などのビジネス特性を踏まえて脅威アクターを絞り込み |
| 要件概要 | 産業セクター（業種）ごとの脅威状況を集計・可視化し、セクター固有の脅威傾向を分析する |
| 具体的な機能 | - 10業種の産業セクター分類（金融, テクノロジー, 製造, 医療, エネルギー, 政府・公共, 通信, 教育, 小売, 運輸） |
|  | - 脅威アイテムへのセクタータグ自動付与（キーワードマッチング） |
|  | - セクター別の深刻度分布・前月比較ヒートマップ |
|  | - セクター別の脅威カテゴリ構成分析（AI-as-Target等の比率） |
|  | - セクター別のトレンド時系列分析 |
|  | - セクタードリルダウン（当該セクターのアクター, TTPs, アイテム一覧） |
| 優先度 | P1（ダッシュボードと同時実装） |
| 現状ギャップ | 未実装。データモデルにセクター概念なし |
| デザイン仕様 | design-requirements-v2.md §2.2 |
| 関連要件 | BR-G（ダッシュボード）のナビ・フィルターと連動、BR-C（個社カスタマイズ）の基盤となる |

## 3. 要件の優先マトリクス

PwCサービスの機能を、本プロジェクトの段階に合わせて優先順位付けする。

### P1（Phase 2 で実装 — プロダクト価値の核）

| BR# | 要件 | 理由 |
|---|---|---|
| BR-G | リアルタイムダッシュボード | ユーザーに価値を見せる最初の接点 |
| BR-K | 産業セクター別脅威分析【v2追加】 | セクター視点の分析はユーザーの「自分ごと化」を促し、BR-Cの基盤となる |
| BR-A | 脅威アクター追跡 | PwCとの最大の差別化を目指す要素 |
| BR-B | TTPs 構造化管理 | ATT&CK 連携はインテリジェンスの質を決定づける |
| BR-E | IoC 管理・配信 | 実務的な価値が最も直接的 |
| BR-J | レポーティング | ユーザーの定期利用を促すコア機能 |

### P2（Phase 3 で実装 — 差別化と高度化）

| BR# | 要件 | 理由 |
|---|---|---|
| BR-C | 個社カスタマイズ | SaaS化・収益化に直結。BR-Kのセクター分類を基盤として発展 |
| BR-D | 検知ルール管理 | 実運用に必要だが、SIEM連携が前提 |
| BR-I | 対応手順管理 | 運用現場での実用性を高める |

### P3（Phase 4 で実装 — 先進機能）

| BR# | 要件 | 理由 |
|---|---|---|
| BR-F | アクティブディフェンス | 高度な分析基盤が前提。AI/LLM活用の象徴的機能 |
| BR-H | 地政学リスク連関 | 独自のデータソースとアナリストの知見が必要 |
