# PROJECT.md — AI Cyber Threat Intelligence Collector

## 1. Why（なぜこのプロジェクトが必要か）

### 背景

AI技術の急速な普及に伴い、AI固有のセキュリティリスクが増大している。
具体的には以下のような新しい脅威領域が生まれている:

- **AIシステムへの攻撃**: プロンプトインジェクション、モデルポイズニング、敵対的入力、モデル逆転・メンバーシップ推定
- **AIを悪用した攻撃**: ディープフェイク、AI生成マルウェア、自動フィッシング、ボイスクローニング
- **AIインフラの脆弱性**: ML フレームワーク（TensorFlow, PyTorch等）やLLMツールチェーン（LangChain等）の脆弱性
- **AIエコシステムの脅威**: GPU クラスタ、クラウドML基盤、ベクトルDBなどの周辺基盤への攻撃
- **AIによる物理空間への脅威**: 自動運転・産業ロボット・医療AIデバイス等の物理系AIシステムへの攻撃（生命安全に直結）
- **AIサプライチェーンへの脅威**: モデルリポジトリへの汚染、学習データのポイズニング、悪意ある事前学習済みモデルの配布
- **自律AIエージェントへの脅威**: エージェントが外部ツールを操作する構成を悪用した攻撃（エージェント経由のSSRF、ツール呼び出し乗っ取り等）

しかし、これらの情報は複数のソースに分散しており、
AI×サイバーセキュリティに特化した統合的な脅威インテリジェンスを
効率的に収集・分析する手段は十分に存在しない。

### ビジョン

AI×サイバーセキュリティ領域の脅威情報を、
**網羅的に収集し、構造化・分類し、実用的なインテリジェンスとして提供する**プラットフォームを構築する。

### 現在のフェーズ

**プレビジネス段階** — MVP の機能検証と市場理解を優先。
AI スタッフ（`/ba`, `/sde`）とのチーム体制で開発を進行中。

### 対象ユーザー（想定）

→ 詳細は `docs/business/user-personas.md` を参照

- AI/MLシステムを開発・運用するセキュリティチーム
- サイバー脅威インテリジェンス（CTI）アナリスト
- AIセキュリティの研究者
- CISO / セキュリティマネジメント層

### 解決する課題

| # | 課題 | 本プロジェクトの解決策 |
|---|------|----------------------|
| 1 | AI関連の脅威情報が複数ソースに分散 | 6つ以上のソースから自動収集し一元化 |
| 2 | 一般的なCVEとAI固有の脅威が混在 | MITRE ATLAS / ATT&CK ベースの分類で AI 脅威を自動識別 |
| 3 | 情報量が多く重要な脅威の見落とし | 深刻度・カテゴリ別のフィルタリングと優先度付け |
| 4 | 脅威トレンドの把握が手作業に依存 | 日次サマリーとトレンド分析を自動生成 |

## 4. 用語定義

| 用語 | 定義 |
|------|------|
| MITRE ATLAS | AI システムへの敵対的脅威を体系化したフレームワーク |
| MITRE ATT&CK | サイバー攻撃の戦術・技術を体系化したナレッジベース |
| CVE | 公開された脅威の共通識別子（Common Vulnerabilities and Exposures） |
| CVSS | 脆弱性の深刻度スコアリングシステム（0.0〜10.0） |
| KEV | CISA が管理する、実際に悪用が確認された脆弱性カタログ |
| STIX/TAXII | 脅威情報を機械可読形式で交換するための標準規格 |
| TLP | 情報共有の範囲を制御するプロトコル（Traffic Light Protocol） |
| AI-as-Target | AI システム自体が攻撃対象となる脅威（プロンプトインジェクション、モデルポイズニング、敵対的入力、モデル逆転等） |
| AI-as-Weapon | AI 技術が攻撃ツールとして悪用される脅威（ディープフェイク、AI生成マルウェア、自動フィッシング等） |
| AI-Enabled | AI の基盤技術・インフラに存在する脆弱性（MLフレームワーク CVE、LLMツールチェーン脆弱性等） |
| AI-Adjacent | AI エコシステムの周辺基盤に関連する脅威（GPU クラスタ、クラウドML、ベクトルDB等） |
| AI-Physical | AI が制御する物理系システムへの脅威（自動運転、産業ロボット、医療AIデバイス、ドローン等）。生命安全リスクを伴う |
| AI-Supply-Chain | AIモデル・データのサプライチェーンへの脅威（汚染済みモデルの配布、学習データポイズニング、悪意ある依存パッケージ等） |
| AI-Agentic | 自律AIエージェントへの脅威（ツール呼び出し乗っ取り、エージェント経由のSSRF、マルチエージェント間操作等） |
